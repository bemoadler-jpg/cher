БАЗА ЗНАНИЙ

// ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ · ТЕКСТ И ВИДЕО

🛰️ ГЕОИНФОРМАЦИОННЫЕ ТЕХНОЛОГИИ
Как правильно установить телематическое оборудование?
ВИДЕО▼
Перед установкой убедитесь в наличии оборудования на вверенном складе. Проведите диагностику устройства, зафиксируйте серийный номер в отчётности. Подключение выполняется по схеме из регламента ГИСИТ. После установки — тест сигнала GPS и передача данных в ПО мониторинга.
▶
ИНСТРУКЦИЯ ПО УСТАНОВКЕ · 6:20
Как вести отчётность по вверенному складу?
ТЕКСТ▼
Ежедневно фиксируйте приход/расход оборудования. Остатки сверяйте с данными в 1С. Отчёт подаётся руководителю каждую пятницу до 17:00 в установленном формате Excel. При расхождении — немедленно уведомите ведущего технического специалиста.
Как обработать заявку по навигации в зоне ответственности?
ТЕКСТ▼
Заявки поступают через Яндекс.Трекер. Примите заявку, присвойте статус «В работе». Выезд на объект — в течение 24 часов. По итогу — акт приёмки, который согласовывает ведущий технический специалист. Закройте заявку в трекере с комментарием.
🚛 АВТОМАТИЗАЦИЯ ТРАНСПОРТНОЙ ЛОГИСТИКИ
Как работать с расхождениями пробегов и топлива?
ВИДЕО▼
Открываете отчёт «Расхождение пробегов» в ПО мониторинга. Сверяете с данными путевых листов в 1С АСУ АТП. Расхождение свыше 5% — фиксируется как инцидент. Сим-карты проверяются в ЛК поставщика связи, при необходимости — активация через ЭДО.
▶
РАБОТА С ОТЧЁТАМИ АТП · 8:45
Как поддерживать пользователей системы 1С АСУ АТП?
ТЕКСТ▼
Консультации пользователей проводятся по запросу через почтовый ящик АТЛ. Права доступа корректируются через администраторский раздел 1С. CR (запросы на доработку) формируются по шаблону и согласовываются с руководителем. Все обращения фиксируются в Яндекс.Трекере.
Как моделировать бизнес-процессы AS IS / TO BE?
ВИДЕО▼
Используются нотации BPMN, UML, EPC или IDEF0 — выбор зависит от сложности процесса. AS IS — текущее состояние, TO BE — целевое. Схемы согласовываются с заказчиком, затем передаются разработчикам в виде функциональных требований. Итоговые материалы оформляются в PowerPoint для руководства.
▶
НОТАЦИИ BPMN/UML · 12:30
📡 ЕДИНЫЙ СИТУАЦИОННЫЙ ЦЕНТР
Как реагировать на тревожные события при транспортировке?
ВИДЕО▼
При получении тревожного сигнала в системе мониторинга — немедленно уточните ситуацию у водителя по телефону. Зафиксируйте событие в журнале инцидентов. При нештатной ситуации уведомите смежные службы и старшего специалиста. Все данные по маршруту доступны в системе видеомониторинга.
▶
АЛГОРИТМ РЕАГИРОВАНИЯ · 5:10
Как формировать отчётность в ЕСЦ?
ТЕКСТ▼
Статичные отчёты формируются ежесуточно по установленным шаблонам. Отчёты по запросу — в течение 2 часов с момента поступления запроса. Данные берутся из систем мониторинга, АСК ТТ и 1С. Анализ нарушений — еженедельно, итоги передаются консультанту ЕСЦ.
